## ADDED Requirements

### Requirement: 项目元数据卡片
系统SHALL在项目详情页顶部以紧凑卡片形式展示项目背景信息。

#### Scenario: 项目头部信息展示
- **WHEN** 用户进入项目详情页
- **THEN** 顶部显示项目元数据卡片
- **AND** 包含项目名称、合作方、开发人员、测试人员等关键信息
- **AND** 信息布局应紧凑美观，便于快速扫视

#### Scenario: 项目名称编辑
- **WHEN** 用户点击项目名称
- **THEN** 进入编辑模式
- **AND** 支持直接修改项目名称
- **AND** 修改后立即保存并更新显示

#### Scenario: 合作方信息管理
- **WHEN** 用户需要配置合作方信息时
- **THEN** 支持输入合作方名称（如"360金条"）
- **AND** 支持添加多个合作方
- **AND** 合作方信息应在悬浮提示中显示

#### Scenario: 联调群链接配置
- **WHEN** 用户配置联调群信息时
- **THEN** 支持插入飞书/企微/钉钉群链接
- **AND** 点击链接可直接跳转到对应群组
- **AND** 支持配置多个群组链接

#### Scenario: 对方联系人管理
- **WHEN** 用户添加对方联系人时
- **THEN** 支持输入姓名、角色、联系方式
- **AND** 格式如"张三 (产品) - 138xxxx"
- **AND** 支持添加多个联系人

#### Scenario: 备注信息编辑
- **WHEN** 用户需要记录特殊信息时
- **THEN** 提供多行文本框输入备注
- **AND** 支持记录鉴权账号、测试环境地址等信息
- **AND** 备注内容应在需要时易于查看

### Requirement: 执行甘特图
系统SHALL提供可编辑的执行甘特图，支持PM日常操作。

#### Scenario: 甘特图列配置
- **WHEN** 执行甘特图加载时
- **THEN** 显示任务名、负责人（头像）、状态、进度条、当日进展等列
- **AND** 列宽应自适应内容长度
- **AND** 支持自定义列显示顺序

#### Scenario: 双击编辑功能
- **WHEN** 用户双击任务行时
- **THEN** 进入编辑模式
- **AND** 支持修改进度（0-100%）、状态、备注等信息
- **AND** 编辑界面应简洁直观

#### Scenario: 实时保存机制
- **WHEN** 用户完成编辑并保存时
- **THEN** 立即更新甘特图显示
- **AND** 无需审批流程，实现极速模式
- **AND** 保存成功后显示确认提示

#### Scenario: 状态管理
- **WHEN** 用户修改任务状态时
- **THEN** 支持选择正常、阻塞、已解决等状态
- **AND** 不同状态应有视觉区分
- **AND** 状态变更应实时反映到甘特图

#### Scenario: 当日进展记录
- **WHEN** 用户记录当日进展时
- **THEN** 提供文本框输入进展描述
- **AND** 支持快速输入常用进展模板
- **AND** 进展信息应在甘特图中适当显示
