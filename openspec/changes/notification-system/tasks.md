# 提醒系统开发任务列表

## 任务1：定义提醒相关类型

**描述**：在types文件中添加提醒相关的类型定义，包括提醒项、提醒状态等

**具体要求**：
1. 添加NotificationItem类型，包含id、taskName、taskId、remindTime、createdAt等字段
2. 添加NotificationStatus类型，包含pending、confirmed等状态
3. 更新Project类型，确保包含startDate字段

**验证方法**：
- 类型定义符合TypeScript语法规范
- 类型定义覆盖提醒系统所需的所有字段
- 类型定义与现有类型兼容

## 任务2：实现提醒状态管理

**描述**：创建提醒系统的状态管理文件，管理提醒列表、未读数量等状态

**具体要求**：
1. 使用现有状态管理方案（如zustand）
2. 实现提醒列表的添加、删除、确认等操作
3. 实现未读提醒数量的计算
4. 实现提醒项的状态更新

**验证方法**：
- 状态管理功能正常工作
- 提醒列表可以正确添加、删除和更新
- 未读数量计算准确
- 状态变更时组件能够及时响应

## 任务3：实现提醒触发工具函数

**描述**：创建提醒触发逻辑和时间比较函数

**具体要求**：
1. 实现时间比较函数，检查当前时间是否到达任务开始时间
2. 实现提醒触发逻辑，当时间到达时添加提醒
3. 实现定时器管理，定期检查任务开始时间
4. 实现定时器的清理功能，避免内存泄漏

**验证方法**：
- 时间比较函数计算准确
- 到达任务开始时间时自动添加提醒
- 定时器正常工作，不影响系统性能
- 组件卸载时定时器正确清理

## 任务4：实现提醒列表下拉组件

**描述**：开发提醒列表的下拉组件，用于展示所有待处理的提醒事项

**具体要求**：
1. 使用Ant Design的Dropdown组件实现
2. 支持显示/隐藏动画效果
3. 实现提醒列表的滚动功能
4. 实现空状态提示
5. 支持点击外部区域关闭

**验证方法**：
- 提醒列表能正常显示和隐藏
- 动画效果流畅自然
- 支持滚动查看多条提醒
- 空状态提示显示正确
- 点击外部区域能正确关闭

## 任务5：实现单个提醒项组件

**描述**：开发单个提醒项组件，包含任务名称、提醒时间和确认按钮

**具体要求**：
1. 显示任务名称、提醒时间等信息
2. 添加确认按钮，点击后移除该提醒
3. 实现确认操作的即时反馈
4. 样式符合现有UI设计规范

**验证方法**：
- 提醒项信息显示完整
- 点击确认按钮后立即移除该提醒
- 确认操作有视觉反馈
- 样式与现有UI保持一致

## 任务6：修改铃铛图标交互

**描述**：更新布局组件中的铃铛图标，添加点击交互功能

**具体要求**：
1. 为铃铛图标添加点击事件处理
2. 集成提醒列表下拉组件
3. 实时更新未读提醒数量
4. 实现铃铛图标的悬停效果

**验证方法**：
- 点击铃铛图标能正常显示/隐藏提醒列表
- 未读提醒数量显示准确
- 悬停效果符合设计规范
- 交互流畅无卡顿

**描述**：将提醒系统集成到应用中，确保各组件协同工作

**具体要求**：

2. 确保提醒系统与现有状态管理无缝集成
3. 确保提醒系统在多页面间正常工作
4. 实现组件卸载时的资源清理

**验证方法**：
- 应用启动时提醒系统正常初始化
- 提醒系统与现有状态管理兼容
- 在不同页面间提醒系统功能正常
- 组件卸载时资源正确清理
