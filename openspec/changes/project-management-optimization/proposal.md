# 项目管理模块优化与修正提案

## Context
当前项目管理系统存在以下问题：
1. 甘特图初始化时未自动定位到当天日期，用户需要手动滚动查找今日任务
2. 项目管理模块包含了设计规范中不应出现的"集数"和"媒体"相关内容
3. 项目管理界面无法完全编辑项目所有信息
4. 任务类型可能缺失关键的开发阶段和测试阶段

## Scope
- **甘特图组件**：`src/components/MasterGantt/index.tsx`
- **项目管理页面**：`src/pages/ProjectManagement/index.tsx`
- **项目编辑模态框**：`src/components/ProjectEditModal/index.tsx`
- **数据模型**：`src/types/index.ts`
- **状态管理**：`src/store/useStore.ts`

## Behavior

### 1. 甘特图优化
- **当前行为**：甘特图初始化时显示当前月份的完整视图，不自动滚动到今天
- **期望行为**：
  - 甘特图初始化时自动滚动到今天的日期位置
  - 确保今日及后续任务清晰可见
  - 保持现有的月份切换功能不变

### 2. 项目管理模块修正

#### 2.1 移除"集数"和"媒体"相关内容
- **当前行为**：项目管理界面显示"导演"、"计划集数"、"已完成集数"、"拍摄中集数"等剧集相关内容
- **期望行为**：
  - 移除所有与"集数"、"媒体"相关的UI展示
  - 移除相关的业务逻辑处理
  - 确保数据模型不包含这些字段
  - 项目管理应专注于通用项目管理，而非特定的剧集管理

#### 2.2 修复设计偏离问题
- **当前行为**：由于添加了"集数"内容，项目管理模块偏离了原始设计规范
- **期望行为**：
  - 恢复项目管理模块的原始设计结构
  - 确保功能和数据结构与设计规范一致
  - 移除所有示例参考内容

#### 2.3 完善任务类型
- **当前行为**：现有任务类型可能不完整，缺少关键的开发和测试阶段
- **期望行为**：
  - 确保包含完整的开发阶段和测试阶段
  - 根据原始设计规范添加相应的任务阶段分类
  - 配置相关属性

### 3. 项目编辑功能修复
- **当前行为**：项目管理界面无法完全编辑项目所有信息
- **期望行为**：
  - 支持修改项目的各个字段
  - 支持修改项目排期
  - 确保编辑功能正常工作

## Risks
- 移除"集数"内容可能影响现有数据，但根据规范这些是示例内容，不应包含在实际项目中
- 甘特图自动滚动功能可能在不同屏幕尺寸下表现不一致
- 修改项目编辑功能可能影响现有的表单验证逻辑

## 验收标准
1. 甘特图初始化时自动滚动到今天的日期
2. 项目管理界面不再显示任何与"集数"、"媒体"相关的内容
3. 项目管理模块功能和数据结构与原始设计规范一致
4. 所有项目字段均可编辑，包括排期
5. 新增的任务阶段功能正常可用
6. 系统运行无错误，功能正常
