# 项目管理模块优化与修正任务分解

## 1. 甘特图自动滚动到今天

### 任务描述
实现甘特图初始化时自动滚动到今天的日期位置，确保今日及后续任务清晰可见。

### 实现步骤
1. 在 `MasterGantt` 组件中添加 `useEffect` 钩子
2. 计算今天日期在时间轴中的偏移量
3. 使用 `containerRef` 实现自动滚动
4. 确保滚动功能在组件加载和日期变化时都能正常工作

### 验证方法
- 甘特图加载时，今日日期自动居中或靠近左侧显示
- 今天及后续任务清晰可见
- 月份切换后，再次点击"今天"按钮能正确滚动到当前日期

## 2. 移除项目管理界面中的"集数"和"媒体"内容

### 任务描述
移除项目管理界面中所有与"集数"、"媒体"相关的内容，包括UI展示和业务逻辑。

### 实现步骤
1. 修改 `ProjectManagement` 组件的 `renderListView` 方法
2. 移除"导演"、"计划集数"、"已完成集数"、"拍摄中集数"等列
3. 替换为通用项目管理字段
4. 修改卡片视图中相关的描述文字
5. 确保过滤和排序功能不受影响

### 验证方法
- 项目管理列表视图不再显示任何与"集数"相关的列
- 项目管理页面描述文字从"管理剧集拍摄进度与状态"改为通用项目管理描述
- 所有项目管理功能正常工作

## 3. 修复项目管理界面的编辑功能

### 任务描述
确保项目管理界面能够编辑项目的所有信息，包括排期。

### 实现步骤
1. 检查 `ProjectEditModal` 组件中的表单字段
2. 确保所有项目字段都在表单中可见且可编辑
3. 修复可能存在的表单验证问题
4. 确保编辑后的数据能正确保存到状态管理

### 验证方法
- 所有项目字段（包括名称、状态、进度、日期、负责人等）均可编辑
- 编辑后的数据能正确显示在甘特图和项目列表中
- 表单验证正常工作，必填字段未填写时给出提示

## 4. 完善任务类型配置

### 任务描述
检查并完善任务类型配置，确保包含完整的开发阶段和测试阶段。

### 实现步骤
1. 检查 `useStore` 中的 `taskTypes` 配置
2. 根据设计规范添加缺失的任务类型
3. 确保每个任务类型都有正确的ID、名称、颜色和启用状态

### 验证方法
- 任务类型包含完整的开发和测试阶段
- 每个任务类型都有唯一的ID和合适的颜色
- 任务类型配置符合设计规范

## 5. 测试与验证

### 任务描述

- 所有功能符合设计规范
