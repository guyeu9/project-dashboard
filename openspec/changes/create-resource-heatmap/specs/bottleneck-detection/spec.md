## ADDED Requirements

### Requirement: 瓶颈识别算法
系统SHALL自动识别人员过载情况，帮助PM发现资源瓶颈。

#### Scenario: 过载检测算法
- **WHEN** 系统分析人员任务分配时
- **THEN** 计算每个人员的并行任务负载
- **AND** 根据预设阈值判断是否过载
- **AND** 过载人员应在热力图中标红显示
- **AND** 算法应考虑任务类型和复杂度权重

#### Scenario: 阈值配置管理
- **WHEN** 管理员需要调整瓶颈判断标准时
- **THEN** 支持设置不同级别的任务数量阈值
- **AND** 支持按人员级别设置不同阈值
- **AND** 支持按任务类型设置权重系数
- **AND** 配置变更应实时生效

#### Scenario: 空闲资源识别
- **WHEN** 系统分析人员工作负载时
- **THEN** 识别任务数量较少的人员
- **AND** 在热力图中用特殊颜色标识空闲状态
- **AND** 提供空闲时间段的具体信息
- **AND** 支持按技能标签筛选空闲人员

### Requirement: 瓶颈预警功能
系统SHALL提供瓶颈预警和统计报告功能。

#### Scenario: 实时预警提示
- **WHEN** 检测到人员过载时
- **THEN** 在界面顶部显示预警提示
- **AND** 显示过载人员姓名和过载程度
- **AND** 支持一键查看详细过载信息
- **AND** 预警信息应定期更新

#### Scenario: 瓶颈统计报告
- **WHEN** 用户需要了解整体资源状况时
- **THEN** 生成瓶颈统计报告
- **AND** 显示过载人员排行和空闲人员列表
- **AND** 提供资源调配建议
- **AND** 支持导出报告为PDF或Excel格式

#### Scenario: 历史趋势分析
- **WHEN** 用户需要分析资源使用趋势时
- **THEN** 提供历史瓶颈数据分析
- **AND** 显示瓶颈发生的频率和模式
- **AND** 预测未来可能的瓶颈风险
- **AND** 支持生成趋势图表
