# UI 设计标准 - 阳光质感原子化设计系统 (Sunshine Atomic Design System)

## 1. 设计哲学 (Design Philosophy)
本系统旨在通过**阳光色系**、**原子化架构**与**微交互动效**，为项目管理平台提供专业且富有活力的视觉体验。
- **阳光向上**：采用高对比度、高饱和度的阳光色系，传达积极的协作氛围。
- **清晰层级**：通过智能阴影系统与 z-index 策略，构建清晰的界面深度。
- **灵动反馈**：每一处交互都应有丝滑的物理动效反馈。

---

## 2. 基础样式规范 (Design Tokens)

### 2.1 色彩系统 (Color System)
| 类型 | 变量名 | 颜色值 | 预览 | 用途 |
| :--- | :--- | :--- | :--- | :--- |
| **品牌色** | `--primary-color` | `#0066FF` | 活力蓝 | 主按钮、选中状态、进度条 |
| **成功色** | `--success-color` | `#23C967` | 阳光绿 | 完成状态、增长指标 |
| **警告色** | `--warning-color` | `#FF9900` | 暖阳橙 | 风险预警、待办提醒 |
| **错误色** | `--error-color` | `#F84F4F` | 警示红 | 延期状态、删除操作 |
| **信息色** | `--info-color` | `#00D1FF` | 晴空蓝 | 提示信息、辅助说明 |

### 2.2 字体与排版 (Typography)
- **字体族**：`Inter, -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- **字阶规范**：
  - `Display`: 32px (Bold) - 核心指标
  - `Title`: 18px (SemiBold) - 页面/卡片标题
  - `Body`: 14px (Regular) - 正文内容
  - `Caption`: 12px (Regular) - 辅助文字

### 2.3 阴影与景深 (Shadow & Depth)
- **卡片阴影** (`--shadow-card`): `0px 2px 4px rgba(26, 28, 30, 0.02), 0px 12px 32px rgba(26, 28, 30, 0.05)`
- **按钮阴影** (`--shadow-button`): `0px 4px 12px rgba(0, 102, 255, 0.2)`
- **浮窗阴影** (`--shadow-dropdown`): `0px 10px 20px rgba(0, 0, 0, 0.1)`

## 3. 组件规范 (Component Standards)

### 3.1 甘特图布局 (Gantt Layout)
- **单元格宽度**: 统一为 `60px`，确保时间轴与数据格对齐。
- **背景色**: 
  - 工作日: `#ffffff`
  - 周末: `#f6ffed` (阳光绿)
- **今日标记**: 
  - 头部: `#fff1f0` 背景，红色文字。
  - 垂直红线: `2px` 宽度，贯穿整个甘特图。
- **文字标准**: 
  - 所有关键文字（标题、标签、日期）统一使用纯黑色 `#000000`，确保最高对比度。
- **状态颜色 (进度条)**: 
  - 进行中: 淡绿色 `#f6ffed` 背景，`#b7eb8f` 边框，`#389e0d` 文字。
  - 风险: 淡红色 `#fff1f0` 背景，`#ffa39e` 边框，`#cf1322` 文字。
  - 延期: 淡黄色 `#fffbe6` 背景，`#ffe58f` 边框，`#d48806` 文字。
- **任务条样式**: 
  - 采用强对比色（100% 不透明度），白色文字。
  - 圆角 Tag 风格，`14px` 圆角。

### 3.2 表单设计 (Form Design)
- **校验原则**: 遵循“最小化必填”原则。
  - 项目表单: 仅“项目名称”必填。
  - 任务表单: 仅“类型”与“起止日期”必填。
  - 进度记录: 仅“日期”与“状态”必填。
- **默认值**: 非必填项应提供合理的默认值（如风险等级默认为 1，任务负责人默认为“未分配”）。

### 3.3 原子化卡片 (Atomic Card)
- **圆角**: `12px`
- **边框**: `1px solid var(--neutral-border)`
- **交互**: 悬停时向上偏移 `4px`，并增强阴影。
- **状态**: 激活时边框变为 `--primary-color`，背景变为微蓝。

### 3.2 动效曲线 (Motion Curves)
- **基础过渡**: `0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- **入场动画**: `Stagger` 阶梯式加载，延迟 `0.1s`。
- **反馈动画**: 缩放比例 `0.98` (Active) / `1.02` (Hover)。

## 4. 响应式适配 (Responsive Strategy)
- **断点设置**:
  - `Mobile`: 320px - 576px
  - `Tablet`: 577px - 992px
  - `Desktop`: 993px - 1440px
  - `Large`: 1441px - 8K
- **策略**: 移动端单列布局，桌面端多列流式布局，8K 屏保持内容区域居中并增加间距。

---

## 5. 向后兼容与后端建议 (Compatibility & Backend Advice)

### 5.1 兼容性保证
- **API 隔离**: 所有 UI 组件不直接依赖后端字段名，通过 `adapter` 层转换。
- **样式沙箱**: 使用 CSS 变量而非硬编码，支持一键切换回旧版视觉。

### 5.2 后端改动建议
1. **字段扩展**: 建议在项目对象中增加 `riskLevel` (1-5) 字段，以便前端实现更精细的颜色映射。
2. **实时通知**: 引入 WebSocket 协议，配合前端的 `NotificationStore` 实现秒级进度更新。
3. **批量接口**: 针对甘特图操作，提供 `batchUpdateTasks` 接口以减少网络往返。
4. **性能监控**: 建议记录 API 响应耗时，配合前端性能指标进行全链路优化。
